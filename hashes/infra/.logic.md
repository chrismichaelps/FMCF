## @Infra — Senior Algorithm Blueprint

### Migration Algorithm

```
INPUT: running Postgres 16 instance + sentinel DB

1. CONNECT
   - psql -h $PGHOST -p $PGPORT -U $PGUSER -d $PGDATABASE

2. APPLY (in lexicographic order)
   FOR each file in migrations/*.sql:
     - Execute file contents
     - On failure: STOP and surface error (no partial rollback in MVP)

3. VERIFY
   - SELECT table_name FROM information_schema.tables WHERE table_name = 'users'
   - Expected: 1 row
```

### Docker Compose Algorithm

```
1. postgres:16-alpine service
   - port: 5432:5432
   - env: POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD
   - healthcheck: pg_isready
   - volume: pgdata (named)

2. No application containers in compose (API/client run via pnpm)
```

### Negative Logic (Prohibited Paths)

- NEVER hardcode production credentials — .env.example only, .env in .gitignore
- NEVER use migrations with DROP TABLE — additive only for MVP
- NEVER skip the health-check — app must wait for pg_isready before connecting
