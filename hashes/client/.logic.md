## @Client — Senior Algorithm Blueprint

### AuthFlow Algorithm

```
INPUT: user submits { email, password } + mode ('signup' | 'login')

1. VALIDATE (client-side pre-check)
   - email matches /^[^\s@]+@[^\s@]+\.[^\s@]+$/
   - password.length >= 8
   - If invalid → set formError state → STOP

2. CALL API
   - mode === 'signup' → api.signUp(email, password)  → POST /auth/signup
   - mode === 'login'  → api.login(email, password)   → POST /auth/login

3. ON SUCCESS (AuthResponse received)
   - Call useAuthStore.setAuthenticated(authResponse)
   - AuthState transitions to { tag: 'authenticated', user: AuthResponse }
   - App renders Dashboard stub

4. ON FAILURE (ApiError thrown)
   - status 400 → show "Invalid email or password format"
   - status 401 → show "Invalid email or password"
   - status 409 → show "Email already registered"
   - status 503 → show "Service unavailable, try again"
   - default   → show "Unexpected error"
```

### AuthState Machine

```
idle  ──[sign-up/login success]──► authenticated
authenticated  ──[logout]──► idle
```

### Negative Logic (Prohibited Paths)

- NEVER store `token` or `passwordHash` in localStorage (MVP — in-memory only)
- NEVER use `any` type in component props or API responses
- NEVER import Effect runtime in browser context (no @effect/platform imports)
- NEVER catch errors silently — all catch blocks must set visible error state
- NEVER use default exports — all exports are named
- NEVER use `.then()` / `.catch()` chains — use `async/await` with `try/catch` at the edge
