# API Router: Logic Blueprint

**Shard:** `@root/packages/api/src/router.ts`
**State:** `BigInt:0x1`
**Fidelity:** Signature

---

## Algorithm

1. **Decode Request** — `Schema.decodeUnknown(LoginPayload)(body)` inside route handler.
2. **Delegate** — `yield* AuthService.signUp(payload)` or `yield* AuthService.login(payload)`.
3. **Encode Response** — `Schema.encode(AuthResponse)(result)`.
4. **Map Errors** — Centralized `HttpRouter.catchAll` that maps typed errors to HTTP status codes via the `error_http_map` in `.contract.json`.

## Negative Logic (Prohibited Paths)

- ❌ Do NOT place business logic inside route handlers.
- ❌ Do NOT manually construct JSON responses — always use `Schema.encode`.
- ❌ Do NOT use `try/catch` in route handlers — errors flow through `E` channel.
- ❌ Do NOT access `UserRepo` or DB directly from the API layer.
