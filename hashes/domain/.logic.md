# Domain Package: Logic Blueprint

**Shard:** `@root/packages/domain`
**State:** `BigInt:0x1`
**Fidelity:** Signature

---

## Algorithm

1. **Define Branded Types** — Use `Brand.nominal<T>()` to create `UserId` and `HashedPassword`.
2. **Define Base Schemas** — Compose primitive schemas with refinements (email pattern, non-empty string).
3. **Compose Entity Schema** — `UserSchema` = `Struct({ id, email, passwordHash, createdAt })`.
4. **Derive Types** — `type User = Schema.Schema.Type<typeof UserSchema>` (no manual interface duplication).
5. **Define Wire Schemas** — `LoginPayload` (inbound, raw strings), `AuthResponse` (outbound, selective — excludes `passwordHash`).
6. **Re-export** — Single barrel export from `index.ts`.

## Invariants

- `UserSchema.passwordHash` must be `HashedPassword` brand (never `string`).
- `AuthResponse` is a **projection** of `UserSchema` — constructed with `Schema.pick` or as an independent struct. It must structurally exclude `passwordHash`.
- No data transformation logic in this package — schemas only.

## Negative Logic (Prohibited Paths)

- ❌ Do NOT use `Schema.parse` — deprecated API.
- ❌ Do NOT export raw `type User = { id: string; ... }` — derive from Schema only.
- ❌ Do NOT include `passwordHash` in `AuthResponse`.
- ❌ Do NOT import from `auth`, `api`, or `db` packages — domain is the dependency root.
- ❌ Do NOT use `any` for schema fallback.
