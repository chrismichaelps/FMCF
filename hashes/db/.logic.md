# UserRepo: Logic Blueprint

**Shard:** `@root/packages/auth/src/UserRepo.ts`
**State:** `BigInt:0x1`
**Fidelity:** Signature

---

## findByEmail Algorithm

1. Execute parameterized `SELECT * FROM users WHERE email = $1 LIMIT 1`.
2. If row exists → decode through `UserSchema` → return `Option.some(user)`.
3. If no rows → return `Option.none()`.
4. On connection failure → fail with `DbConnectionError`.

## create Algorithm

1. Execute parameterized `INSERT INTO users (email, password_hash) VALUES ($1, $2) RETURNING *`.
2. On success → decode returned row through `UserSchema` → return `User`.
3. On unique constraint violation (Postgres code `23505`) → fail with `DuplicateEmailError`.
4. On other DB error → fail with `DbConnectionError`.

## Negative Logic (Prohibited Paths)

- ❌ Do NOT use raw SQL string interpolation — always use parameterized queries.
- ❌ Do NOT swallow DB errors silently — map to explicit tagged errors.
- ❌ Do NOT return unvalidated rows — always pass through `Schema.decodeUnknown(UserSchema)`.
- ❌ Do NOT implement caching in this layer — that belongs to a future Cache shard.
- ❌ Do NOT import from `@sentinel/api` or `@sentinel/auth` — UserRepo is a dependency root for auth.
